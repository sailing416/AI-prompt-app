Prompt Engineering Guide

The Claude Prompt
Blueprint

A practical template and framework for writing prompts that get exactly what you need from Claude.

1
Set the Role / Persona
What to write

Tell Claude who to act as. This shapes tone, depth, and expertise level.

You are [expert role] with deep experience in [domain]. Your communication style is [tone: concise / detailed / friendly / technical].
✓ Do

You are a senior product manager who specializes in B2B SaaS onboarding flows.

✗ Don't

Be an expert and help me.

2
Provide Project Context Most Important
Background — what Claude needs to know
## Project Background Project name: [name] What it does: [1-2 sentence description] Target audience: [who uses this / who reads this] Current stage: [idea / MVP / live product / revision] Key constraints: [budget / time / tech stack / word count / etc.]
💡 Pro tip The more context you give, the less back-and-forth you need. Paste in relevant docs, code, or notes directly — Claude can read and reason over them.
3
Define the Task Clearly
Action verb + specific deliverable
Task: [Write / Analyze / Fix / Plan / Review / Build / Summarize] Specifically: [one-sentence description of what to produce] Scope: - Include: [what must be in the output] - Exclude: [what to leave out]
✓ Clear

Write a 300-word product description for our landing page. Focus on benefits, not features. Exclude pricing.

✗ Vague

Write something about our product for the website.

4
Specify Output Format
Structure, length, and style
Output format: [bullet list / numbered steps / paragraphs / table / code / JSON / markdown] Length: [word count / number of items / approximate pages] Tone: [formal / casual / persuasive / instructional / conversational] Use headers: [yes / no] Show reasoning: [yes — think step by step / no — just give the result]
For code Specify language, whether to include comments, and preferred code style.
For writing Give a word count range and share 1-2 examples of tone you like.
For analysis Ask Claude to think step by step before giving a final answer.
For lists Say how many items you want and how detailed each should be.
5
Give Examples Optional but powerful
Show what "good" looks like
## Examples Good example: [paste 1-2 examples of the style / format / quality you want] Bad example (optional): [paste what you DON'T want, so Claude avoids it]
Why this works so well Examples calibrate Claude faster than any description. Even one good example cuts misunderstandings by ~70%.
6
Add Rules & Constraints
Hard rules Claude must follow
## Rules - Always [required behavior] - Never [forbidden behavior] - If you're unsure about X, [ask / make a reasonable assumption and say so] - If you can't complete the task, [explain why / do your best]

⚡ Quick Tips That Make a Big Difference
Be specific about length "Around 200 words" works better than "short".
Paste in raw material Give Claude your notes, drafts, or data — it reasons over them well.
Ask for step-by-step thinking For complex tasks, add "think through this step by step" before answering.
Iterate, don't restart Say "refine the tone" or "make it more concise" rather than re-prompting from scratch.
Name what you don't want "Don't use jargon" or "avoid bullet points" is just as useful as saying what you do want.
Ask Claude to ask you End with "If anything is unclear, ask me before starting."
Split big tasks Break complex projects into steps. Do each in its own message.
Share the 'why' Context like "this is for a non-technical audience" changes output quality dramatically.
📖 Real Example — Before & After
✗ Weak prompt

Write me an email about our new feature.

✓ Strong prompt

You are a B2B SaaS copywriter. Project: Taskflow — a project management tool for remote teams. Audience: Existing paid users (non-technical, team leads). New feature: AI-generated weekly summaries of team progress. Task: Write a product announcement email. Include: What the feature does, why it saves time, one-click CTA to try it. Exclude: Technical details, pricing. Format: 3 short paragraphs + subject line. Tone: Warm and enthusiastic but professional. Length: Under 180 words. Never use the word "leverage".
